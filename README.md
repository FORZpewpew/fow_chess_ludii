# Fog of War Chess — AI Research Project

A research project studying AI agents for **Fog of War Chess** (Dark Chess) — an imperfect-information variant of chess where each player sees only the squares their pieces can attack or move to. Implemented in the [Ludii](https://ludii.games/) General Game System (v1.3.14) with eight agents spanning random baselines, MCTS variants, and a PPO-LSTM recurrent neural network.

## What Was Studied

The project investigates how different AI paradigms handle **imperfect information** in a chess-like game:

- **Search agents** (UCT, IS-MCTS, GRAVE): how does explicit uncertainty handling via determinization compare to naïve tree search on the observable state?
- **Alpha-Beta agents** (AB-Heuristic, AB-Learned): can a good hand-crafted or self-play-learned heuristic compensate for the lack of rollouts?
- **Recurrent RL** (PPO-LSTM): can an LSTM policy trained end-to-end via PPO develop an implicit belief state purely from observation sequences and sparse win/loss rewards?
- **Behavioural Cloning pretraining**: does BC pretraining on self-play game histories improve both convergence and final performance of the RL agent?
- **Belief-state analysis**: does the LSTM hidden state encode information about hidden opponent pieces?

## Agents

| # | Name | Slug | Source | Description |
|---|------|------|--------|-------------|
| 1 | **Random** | `random` | Ludii built-in | Uniform random move selection. Performance floor. |
| 2 | **UCT** | `uct` | Ludii built-in | MCTS/UCT on the observable state. 1s/move default. |
| 3 | **IS-MCTS** | `ismcts` | `ISMCTSAgent.java` | 10 determinizations × 200 UCT iterations, plurality voting. |
| 4 | **IS-MCTS v4** | `ismcts_v4` | `ISMCTSAgent.java` | 25 determinizations variant. |
| 5 | **GRAVE** | `grave` | Ludii built-in | MCTS with GRAVE selection. |
| 6 | **GRAVE+MAST** | `grave_mast` | Ludii built-in | GRAVE selection + MAST playout policy. |
| 7 | **PPO-LSTM v4** | `ppo_lstm_v4` | `PPOLSTMAgent.java` + Python | Recurrent actor-critic trained from scratch via PPO. Elo ~1267. |
| 8 | **PPO-LSTM Pretrained v4** | `ppo_lstm_pretrained_v4` | `PPOLSTMAgent.java` + Python | BC pretraining + PPO fine-tuning. Elo ~1283. |

Additional agents in codebase (not in final 8-agent tournament):
- **AB-Heuristic** (`ab_heuristic`): Alpha-Beta + FoW-specific heuristic. Dominates all agents.
- **AB-Learned** (`ab_learned`): Alpha-Beta + self-play learned linear value function.

All agents operate exclusively on the **filtered observable context** — they never access hidden opponent information.

## Quick Start

### Prerequisites

- **Java 17+** (JDK; tested with OpenJDK 26 via asdf)
- **Python 3.9+**
- **PyTorch 2.x** and **JPype1** (for training); standard scipy/numpy for analysis

### Step 1: Compile Java agents

```bash
bash fow_chess_ludii/evaluation/scripts/build.sh
```

This compiles all `.java` source files against `Ludii-1.3.14.jar` and packages them into `fow_chess_ludii/agents/jars/agents.jar`.

Or compile manually:

```bash
javac -cp fow_chess_ludii/Ludii-1.3.14.jar \
      fow_chess_ludii/agents/src/*.java \
      -d fow_chess_ludii/agents/compiled/

jar cf fow_chess_ludii/agents/jars/agents.jar \
    -C fow_chess_ludii/agents/compiled .
```

### Step 2: Set up Python environment (for PPO agents and training)

```bash
cd fow_chess_ludii
python3 -m venv ppo/venv
source ppo/venv/bin/activate
pip install torch numpy jpype1 scipy matplotlib
```

> **Note:** The PPO agents require the venv at `ppo/venv/` because `PPOLSTMAgent.java` hardcodes `ppo/venv/bin/python` as the Python interpreter.

### Step 3: Run a quick test match

```bash
# 10 games, Random vs UCT
java -cp "fow_chess_ludii/Ludii-1.3.14.jar:fow_chess_ludii/agents/jars/agents.jar" \
     agents.EvalRunner \
     --agent1 random \
     --agent2 uct \
     --num-games 10 \
     --output /tmp/test_match.csv
```

### Step 4: Run a match with PPO-LSTM

```bash
# Must be run from fow_chess_ludii/ as working directory so Python subprocess paths resolve
cd fow_chess_ludii

java -cp "Ludii-1.3.14.jar:agents/jars/agents.jar" \
     agents.EvalRunner \
     --agent1 ppo_lstm_v4 \
     --agent2 uct \
     --num-games 20 \
     --output /tmp/ppo_vs_uct.csv
```

### Step 5: Run the full V4 tournament

```bash
bash fow_chess_ludii/evaluation/scripts/run_tournament_v4.sh
```

Results are written to `fow_chess_ludii/evaluation/results_v4/`.

### Step 6: Analyse results

```bash
# Compute Elo ratings
python3 fow_chess_ludii/evaluation/scripts/compute_elo.py

# Statistical significance
python3 fow_chess_ludii/evaluation/scripts/significance.py

# Plot win-rate heatmap and Elo bar chart
python3 fow_chess_ludii/evaluation/scripts/plot_results.py

# Or run all analyses at once
python3 fow_chess_ludii/evaluation/scripts/run_all_analyses.py
```

## Training

### Train PPO-LSTM from scratch

```bash
cd fow_chess_ludii
source ppo/venv/bin/activate

# Full training (400 updates × 16 episodes each)
python3 ppo/train_ppo_lstm.py

# Warm-start from an existing checkpoint
python3 ppo/train_ppo_lstm.py --checkpoint checkpoints/ppo_lstm_v4_policy.pt

# Quick smoke-test
python3 ppo/train_ppo_lstm.py --updates 2 --episodes 2
```

Checkpoints are saved every 25 updates to `checkpoints/ppo_lstm_v4_ckpt_{update}.pt`.  
Final checkpoint: `checkpoints/ppo_lstm_v4_policy.pt`.

### Train PPO-LSTM with BC pretraining

**Step 1:** Generate self-play training data (requires compiled `DataCollector.java`):

```bash
java -cp "Ludii-1.3.14.jar:agents/jars/agents.jar" \
     agents.DataCollector \
     --num-games 500 \
     --output training/results/selfplay_training_data.jsonl
```

**Step 2:** Run BC pretraining + PPO fine-tuning:

```bash
cd fow_chess_ludii
source ppo/venv/bin/activate

# BC only (50 epochs with early stopping)
python3 ppo/pretrain.py

# BC + PPO fine-tuning (recommended)
python3 ppo/pretrain.py --ppo_finetune --ppo_updates 300
```

Final checkpoint: `checkpoints/ppo_lstm_pretrained_v4.pt`.

### Train the Belief Probe

```bash
cd fow_chess_ludii
source ppo/venv/bin/activate

# Generate probe data (records LSTM hidden states + true board)
python3 ppo/generate_probe_data.py

# Train the probe
python3 ppo/train_belief_probe.py

# Evaluate
python3 ppo/eval_belief_probe.py
```

## Evaluation

### EvalRunner CLI reference

```
java -cp "Ludii-1.3.14.jar:agents/jars/agents.jar" agents.EvalRunner \
     --agent1 <slug>           # P1 agent
     --agent2 <slug>           # P2 agent
     --num-games <N>           # Games to play (default: 100)
     --output <path>           # CSV output path (appends if exists)
     --time-per-move <secs>    # Think time per move (default: 1.0)
     --max-moves <N>           # Hard move cap, recorded as draw (default: 300)
     --game <path>             # .lud file path (default: ludii/FoW_Chess.lud)
```

**Output CSV schema:**
```
game_id, agent_p1, agent_p2, winner, num_moves, draw, timestamp_utc
```

### Time-control sensitivity

```bash
# 2s/move tournament
bash evaluation/scripts/run_time_sensitivity.sh 2.0

# 5s/move tournament
bash evaluation/scripts/run_time_sensitivity.sh 5.0
```

### Belief Jaccard logging

```bash
# Run IS-MCTS with Jaccard logging enabled
java -Dfow.belief.log=true \
     -Dfow.belief.logfile=evaluation/results/belief_accuracy.csv \
     -cp "Ludii-1.3.14.jar:agents/jars/agents.jar" \
     agents.EvalRunner \
     --agent1 ismcts --agent2 uct \
     --num-games 50

python3 evaluation/scripts/analyze_belief_accuracy.py
```

## Key Results

| Finding | Detail |
|---|---|
| **AB agents dominate** | AB-Heuristic and AB-Learned achieve ~100% win rate vs. all search and RL agents |
| **UCT beats IS-MCTS** | Counterintuitive: UCT outperforms IS-MCTS because budget-splitting produces shallower searches per determinization |
| **PPO-LSTM beats UCT** | PPO-LSTM v4 (Elo ~1267) and Pretrained v4 (Elo ~1283) both outperform UCT (Elo ~1167) |
| **BC pretraining helps** | BC pretraining improves both convergence speed and final Elo |
| **Rankings stable** | Agent ranking is consistent across 2s and 5s time controls |
| **Emergent belief state** | Belief probe achieves 89.68% accuracy on hidden square classification from LSTM hidden states |
| **Jaccard degrades** | IS-MCTS belief quality: Jaccard ~0.80 early-game → ~0.11 late-game; random placement worsens as more pieces are captured |
